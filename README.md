# My Portfolio Project

Hi everyone! This is my personal portfolio website. I built it from scratch to show my skills and the projects I made as a frontend developer. 

I made this project to practice my HTML, CSS, and JavaScript, and also to learn how to use Git and GitHub properly like real developers.

---

## 1. What's on the Website?
The website has a cool dark theme with a nice lime-green color. It has 5 main parts:
1. **Home:** The first page people see. It has my name, my job title, my photo, and links to download my resume.
2. **About:** A short intro about me and what skills I have.
3. **Services:** Cards that show what I can do (like clean coding and web design) with some cool hover movements when you touch them with the mouse.
4. **Projects:** A place where I showcase the websites I built before.
5. **Contact:** A simple form so people can send me messages, plus links to my social media accounts.

---

## 2. Tech I Used & Making it Responsive
* **HTML5:** I used it to build the main structure of the pages.
* **CSS3:** Used for colors, fonts, and custom variables to make the design clean and add smooth movements.
* **JavaScript:** I wrote some JS code to make the header background change when you scroll down, and to make the mobile menu open and close.
* **Responsive Design:** The website looks great on PC, laptops, tablets, and phones. I used Media Queries so that on mobile screens, the top menu changes into a nice sidebar that slides out.

---

## 3. How I Managed the Project (Git & GitHub)
I used Git commands in the Cmder to organize my work just like teams do in real life:

1. First, I finished the 5 main sections and pushed them to the `main` branch.
2. Then, I opened an **Issue** on GitHub to track my work for making the website responsive.
3. I created a new branch called `feature/design-update` so I could write the responsive code safely without messing up the main code.
4. After finishing, I pushed the branch, opened a **Pull Request (PR)**, and linked it to the issue so it closes automatically.
5. Finally, I went back to the `main` branch, did a **Local Merge** in my Cmder, and pushed everything to GitHub.
